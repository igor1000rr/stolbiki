### 28.04.2026 — Ревизия ТЗ Александра «Тест приложения»

Большой блок правок по обратной связи Александра. Полный список ниже,
сгруппирован по разделам ТЗ.

#### Хедер / Лендинг

- **Victory City → City of Victories** во всех точках интерфейса
  (hero, feature card, FAQ, share, embed, onboarding, SEO, 3D-превью).
  Коммиты: `7635341`, `39c7128`, `b5d2793`, `ded1cee`, `c0d9d3a`,
  `e6af336`, `670cb94`.
- Кнопка «Город побед» под TikTok ведёт в раздел Города в профиле.

#### Раздел PLAY

- **Меню за полосой сворачивания** — `safe-area-inset-bottom` на
  `.native-app .site-content` чтобы нижние пункты More-страницы не
  заходили под native-tabs (фикс был ранее в native.css + game-layout.css).
- **Микродвижения игрового поля** — убраны infinite-анимации создававшие
  «дрожание»: `goldenPulse`, `scoreGlow0/1`, `starPulse`. Свечения остаются
  статичными. `9c703fd`.
- **Игровое поле зафиксировано** — на native (Capacitor) при tab='game'
  блокируется body-overflow, что убирает iOS rubber-band и Android
  bounce, создававшие эффект «окна, которое можно двигать вверх-вниз
  как у браузера». На web — без изменений (там нужен normal scroll).
  `4c0fe27`.
- **Tower Takeover transparent on hover** — кнопка остаётся яркой при
  hover/active, прозрачности больше нет. `33862fd`.
- **Игровое поле:**
  - **Web 10→11 стоек после закрытия первой** — БАГ-ФИКС: `min-height`
    стойки на десктопе (308→332px) и mobile (200→216px) теперь
    рассчитан под 11 чипов с самого старта, layout не прыгает при
    закрытии. `7afff0b`.
  - **Огромные стойки в приложении** — на mobile web уменьшены
    стойки 56→44px, чипы 40×20→32×16. `d0715e1`.
  - **Буквы прилипают к столбцам** — largeText по умолчанию выключен
    в settings, `padding-top` на board увеличен до 32px на mobile
    чтобы золотая ★ не упиралась в потолок. `837a316` + `d0715e1`.
  - **EN: «Вы — blue»** — i18n-перевод стартового сообщения партии и
    подсказок-hints (откат сломанной analyzePosition + проброс lang).
    `c941622`, `47209a9`.
- **Окно победы дёргается + чёрный фон в светлой теме** — `animate=false`
  + native фон `color-mix` вместо хардкоднутого `rgba(13,13,20)`.
  `bd68a52`.
- **Светлая тема по умолчанию** для новых игроков (minimal preset).
  `014cfcf`.
- **Дубль Settings шестерёнки + солнышко→шестерёнка**:
  - Иконка `theme` (компонент Icon.jsx) теперь шестерёнка. `5d4f186`.
  - Изначально удалили видимый `<span>⚙</span>` из GameModeBar
    (`7c6e14a`), но Александр пересмотрел: «после уменьшения стоек
    несимметричность не будет видна — оставим». Шестерёнка возвращена.
    `5a59272`.
  - Основная кнопка Settings: SVG лучей солнца → feather-style gear.
    `de23d5d`.
- **Footer тёмный шрифт** на светлой теме. `36d935c`.

#### Раздел Профиль

- **ELO выходит за границы** — фикс на собственном профиле и на
  публичном профиле. `02087093`.
- **Logout по центру** — кнопка center-aligned. `7635341`.
- **Дубль Snappy в Городе побед** — убран triggerSnappy из Profile.jsx,
  Snappy появляется только из VictoryCity (`variant=anchored`). `5c15222`.
- **Земля города зелёная во всех пресетах** — luminance ночных оттенков
  поднят (night, night_neon, dusk), чтобы зелень не сливалась с тёмным

#### Раздел More

- **Lessons убрать в native** — пункт скрыт на native-платформах
  (на web был убран ранее). `9935ca8`.

#### Проблема 1 — Ход противника непонятен

- **Snappy на ход противника** — событие `enemy_takeover` с 5 фразами
  RU/EN, триггер в Game.jsx confirmTurn. `9409ec0`, `fbe4350`.
- **Разные эффекты у переноса и установки** — Board.jsx detect-логика
  типа последнего хода (transfer/place) на основе изменений stands.
  Стойки подсвечиваются 1.2с разными цветами через CSS-классы:
  `stand-recent-transfer-from` (accent), `stand-recent-transfer-to`
  (green), `stand-recent-place-pN` (цвет игрока). Игрок видит «откуда →
  куда» для переноса и цвет ставившего для установки. `74de87b`.
- **Ход противника происходит быстро** — пауза перед разблокировкой
  следующего хода игрока увеличена с 500мс до 1300мс. Это даёт
  recentAction-подсветке (1.2с) полностью отыграть до того как игрок
  начинает ход. `6a1533b`.

#### Проблема 2 — Не понятно чей ход

- **Стрелочки ▶/◀ под цвет текста** в scoreboard у активного игрока,
  пульсация по направлению хода. `124e858`, `5dd47d2`.

#### Проблема 3 — Обучающая партия

- **TutorialGame подключён** — обучающая партия из 3 шагов с точными
  расстановками из ТЗ (Перехват → Установка → Завершение). Окно
  с подсказками над башнями, поле упрощённое (без счёта/таймеров),
  кнопки как в обычной игре. ФИЧА. `87c9692` (импорт+edgeBack),
  `698af31` (кнопка «Обучение» в More), `8a30fd7` (исходные файлы
  TutorialGame.jsx + tutorial.css — раньше были untracked,
  теперь реально в репо).
- **Snappy «Обучение пройдено. Ну наконец-то.»** — финальная фраза
  из event `tutorial_complete` (точная цитата Александра). `cd8649d`.
- **Snappy «Чужую не трогай!»** — event `tutorial_wrong_target`
  срабатывает на шаге 1 при попытке переноса на стойку с верхом
  другого цвета и на шаге 2 при попытке установки на стойку противника.
  `ddcb58a`.
